# Правила оформления коммитов

Сообщение коммита:
- На **русском языке**
- Одно предложение
- Глагол в **прошедшем времени от первого лица**
- Описывает выполненное действие

**Примеры:**
- «Добавил браузерный клиент для Web Push»
- «Исправил ошибку валидации формы»
- «Переработал архитектуру планировщика»
- «Обновил зависимости до актуальных версий»

## Атомарность

Каждый коммит должен отражать **одно логическое действие**. Если изменения затрагивают несколько несвязанных вещей — разбей на несколько коммитов, стейджя файлы по отдельности.

## Процесс

1. Выполни `git status` и `git diff` чтобы понять что изменилось
2. Определи, можно ли сгруппировать изменения в одно логическое действие. Если нет — создай несколько коммитов
3. Сформулируй сообщение по правилам выше
4. Сделай коммит с указанием себя как соавтора:

```
Co-Authored-By: {название используемой модели} <noreply@anthropic.com>
```
