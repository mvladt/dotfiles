## Именование

- `describe()` — называет тестируемую сущность: компонент, модуль, функцию
- `it()` — описывает ожидаемый результат, не действие

Правильно:
- `describe("API пользователя")`, `describe("Форма входа")`, `describe("Форматирование даты")`
- `it("Возвращает пользователя по ID")`, `it("Выбрасывает ошибку при невалидном email")`

Неправильно:
- `describe("Тесты юзера")`, `describe("Проверки")`, `describe("Тест проверки валидации формы")`
- `it("Тест ошибки")`, `it("Проверка пользователя")`

## Структура теста

- Порядок: Arrange → Act → Assert
- Все зависимости, моки и данные инициализируются внутри `it()` или `beforeEach()`
- Инициализация вне тела теста — только для неизменяемых констант

## Изоляция

- Никакого разделяемого мутируемого состояния между тестами
- Каждый тест независим и может запускаться в любом порядке
- Уникальные значения (логины, имена файлов) генерировать через `crypto.randomUUID()`
